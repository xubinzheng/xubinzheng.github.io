IBM BlueMix mobile data IOS SDK "pod install" error solution

From(https://www.ng.bluemix.net/docs/#starters/mobile/ios/ios.html). When running the "pod install" command, the following error is returned:

[!] Due to the previous naïve CocoaPods resolver, you were using a pre-release version of IBMBluemix, without explicitly asking for a pre-release version, which now leads to a conflict. Please decide to either use that pre-release version by adding the version requirement to your Podfile (e.g. pod 'IBMBluemix', '= 1.0.0.20140707-1250') or revert to a stable version by running pod update IBMBluemix.

However, running pod update IBMBluemix results in:

[!] Due to the previous naïve CocoaPods resolver, you were using a pre-release version of IBMFileSync, without explicitly asking for a pre-release version, which now leads to a conflict. Please decide to either use that pre-release version by adding the version requirement to your Podfile (e.g. pod 'IBMFileSync', '= 1.0.0.20140707-1250') or revert to a stable version by running pod update IBMFileSync.

and trying pod update IBMFileSync results in it looping back to the error about updating IBMBluemix pod.

It seems the latest update to Cocoapods is causing this issue. I was forced to revert to an earlier version to allow the dependencies to install correctly.

If you have the latest version installed you will have to call:

 [sudo] gem uninstall cocoapods

I was forced to uninstall the other gems as well:

 [sudo] gem uninstall cocoapods-core
 [sudo] gem uninstall cocoapods-downloader  
 [sudo] gem uninstall cocoapods-plugins
 [sudo] gem uninstall cocoapods-trunk
 [sudo] gem uninstall cocoapods-try

Then you can install any version below 0.35.0. I used 0.33.1 but 0.34 is supported as well:

 [sudo] gem install cocoapods -v 0.33.1

After done the previous steps, the Bluemix mobile data service SDK should be work.

https://www.ng.bluemix.net/docs/#starters/mobile/ios/ios.html
