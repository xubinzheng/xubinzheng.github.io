I just found something wrong about SDK IOS starter.

Add the Mobile Cloud Services SDK to your iOS app The SDK contains the client code to access all of the mobile services. Both SDKs are subject to the license agreement contained within the archive.

Install the SDK and dependencies with CocoaPods. From the folder where you have downloaded your sample, run the following command:

$ pod install

But I really suggest if someone want to install the SKD, use the following command firstly instead previous one

$ pod update

Because sometimes due to the previous naive CocoaPods resolver, someone were using a pre-release version of 'IBMBluemix', without explicity asking for a pre-release version, which now leads to a conflict. So use this command to check firstly will be really helpful.

https://www.ng.bluemix.net/docs/#starters/mobile/ios/ios.html
